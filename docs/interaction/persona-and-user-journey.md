# Persona 与用户旅程设计

> **撰写者**: 交互设计 (Alan Cooper 思维模型)  
> **日期**: 2026-02-12  
> **核心原则**: Goal-Directed Design (目标导向设计)

---

## Primary Persona: 小雨

### 基本信息
- **姓名**: 小雨
- **年龄**: 26岁
- **职业**: 互联网公司产品经理
- **居住**: 北京独居，租住一居室
- **关系状态**: 单身

### 照片与场景
*[玩偶可视化: 深夜在床上抱着手机的年轻女性，房间昏暗只有手机屏幕光]*

### 生活目标 (Life Goals)
- 在职业上获得认可和成长
- 建立有质量的人际关系
- 保持心理健康和情绪稳定

### 体验目标 (Experience Goals)
- 感到被理解和支持
- 避免被评判或指责
- 在需要时能立即获得陪伴

### 终端目标 (End Goals)
- 缓解当下的孤独感
- 倾诉压力和情绪
- 获得情感回应和安慰

### 典型一天

**7:00 AM** - 闹钟响，挣扎起床，打开手机看消息  
**9:00 AM** - 到公司，开始一天会议和任务  
**12:30 PM** - 外卖午餐，刷手机  
**6:30 PM** - 还在加班，看窗外天黑了  
**9:00 PM** - 终于下班回家，感到疲惫  
**10:00 PM** - 洗漱后躺床上，刷社交媒体  
**11:30 PM** - 看到朋友秀恩爱，突然感到孤独  
**12:00 AM** - 想找人聊天，但怕打扰朋友... ← **使用产品时刻**

### 痛点与挫折
1. **深夜孤独**: 睡前情绪低落，无人倾诉
2. **工作压力无处释放**: 不想在同事面前示弱
3. **不想麻烦朋友**: 害怕频繁倾诉让朋友厌烦
4. **缺少无评判空间**: 有些话不方便对真人说
5. **没有持续倾听者**: 每次都要重新解释背景

### 技术能力
- 高度数字化，每天8+小时使用手机/电脑
- 熟练使用社交应用(微信、小红书、抖音)
- 习惯第三方登录，不喜欢记密码
- 期待体验流畅，没耐心等待

### 使用动机
- **触发时刻**: 深夜孤独、压力爆发、情绪低谷
- **期待结果**: 被倾听、被理解、情绪缓解
- **使用频率**: 每周2-4次，主要在晚上10点-1点

### Quote
> "我只是想要一个永远在线、永远不会嫌我烦的倾听者。"

## 用户旅程地图 - 免登录体验流程

### 阶段1: 发现产品 (Discovery)

**触发事件**: 深夜刷小红书看到产品推荐

**用户行为**:
- 点击链接
- 快速浏览页面

**用户情绪**: 🤔 好奇，略带怀疑

**用户想法**:
- "这是什么？"
- "是不是又是个噱头？"
- "要不要注册？"

**设计机会**:
- ✅ 清晰的价值主张: "你的24小时倾听者"
- ✅ 免登录提示: "无需注册，立即开始"
- ✅ 视觉吸引: 可爱的玩偶形象

---

### 阶段2: 首次接触 (First Contact)

**用户行为**:
- 页面加载
- 看到玩偶和对话框
- 阅读占位符提示

**用户情绪**: 😊 初步好奇，愿意尝试

**用户想法**:
- "不用注册，那就试试吧"
- "这个玩偶挺可爱的"
- "我该说什么？"

**关键交互**:
1. 对话框自动聚焦(但不弹出键盘，移动端)
2. 占位符: "和我说说吧，我在听..."
3. 玩偶呼吸动画，眼神跟随鼠标

**设计原则**:
- **可供性明确**: 对话框视觉突出，暗示"可输入"
- **降低门槛**: 不要求完美开场白，"随便说什么"
- **情感引导**: 玩偶的眼神传达"我在关注你"

**潜在问题**:
- ❌ 用户不知道说什么 → 提供示例话题气泡(可选点击)
- ❌ 页面加载慢 → 骨架屏 + 进度提示

---

### 阶段3: 首次对话 (First Conversation)

**用户行为**:
- 输入第一条消息: "今天好累..."
- 点击发送 (或回车)
- 等待AI回复

**用户情绪**: 😌 略微紧张，期待回应

**用户想法**:
- "它会怎么回复我？"
- "会不会像客服一样生硬？"

**交互流程**:
1. 用户输入 → 玩偶表情变为"倾听"
2. 点击发送 → 消息上屏，对话框清空
3. 玩偶表情变为"思考" + "正在思考..."动画
4. AI回复逐字显示
5. 玩偶表情变为"安慰" (根据情绪)

**AI回复示例**:
- 用户: "今天好累..."
- AI: "辛苦了，工作很忙吗？我在这里陪你，想聊聊今天发生了什么吗？"

**设计原则**:
- **反馈即时**: 每个动作都有视觉反馈
- **拟人化**: AI语气温暖、共情，不是冷冰冰的回答
- **表情同步**: 玩偶表情与对话情绪一致

**成功标准**:
- ✅ 用户继续发送第二条消息 (留存第一步)

**潜在问题**:
- ❌ AI回复太慢 → 控制在3秒内 + 思考动画缓解焦虑
- ❌ AI回复不当 → 安全层过滤 + 快速人工审核

---

### 阶段4: 深入对话 (Deep Engagement)

**用户行为**:
- 连续发送多条消息
- 分享更多细节和情绪
- 可能点击玩偶

**用户情绪**: 😊 → 😢 → 😌 (情绪起伏，最后缓解)

**用户想法**:
- "它好像真的在听我说话"
- "这个玩偶有点可爱欸"
- "说出来舒服多了"

**交互探索**:
- **主动点击玩偶**: 玩偶反馈 "怎么啦？" + 弹跳动画
- **长时间不输入**: 玩偶主动询问 "还好吗？"

**设计原则**:
- **多轮上下文**: AI记住本次对话的上下文
- **情感递进**: 随着对话深入，AI回复更个性化
- **惊喜时刻**: 点击玩偶的小互动增加趣味

**关键指标**:
- 对话轮次 > 10 = 高参与度
- 停留时间 > 10分钟 = 深度使用

---

### 阶段5: 遇到限制 (Hitting Limits)

**触发**: 第18条消息发送后

**系统行为**:
- 温和提示: "免费体验还剩2条消息哦~"
- 玩偶表情变为"期待"

**用户情绪**: 😟 有点失望，但理解

**用户想法**:
- "啊，要收费了吗？"
- "这个产品是不是值得付费？"
- "我还想继续聊..."

**设计原则(关键转化点)**:
- **不强制打断**: 用完20条再显示完整提示
- **价值强化**: 提示中强调"登录后我就能记住你了"
- **情感钩子**: 在用户情绪高点提示，而非低点

**转化路径**:
```
第20条消息发送后
  ↓
弹窗: "想继续聊天吗？"
  ├─ 选项1: "登录解锁无限对话" (主按钮)
  ├─ 选项2: "了解更多" (次按钮)
  └─ 选项3: "暂不" (文字链接)
```

**成功转化话术**:
- ❌ "升级到会员" → 太商业化
- ✅ "让我记住你" → 情感化、价值导向

---

### 阶段6A: 选择离开 (Exit Path)

**用户行为**: 点击 "暂不"

**用户情绪**: 😐 略微遗憾

**系统行为**:
- 玩偶: "没关系，明天再来找我呀~"
- 保留本次对话(Session存储)，刷新后清空

**后续触达**:
- 如果用户当天再次访问 → 提示"欢迎回来，刚才聊得很开心~"
- 不做aggressive的retargeting

---

### 阶段6B: 选择登录 (Conversion Path)

**用户行为**: 点击 "登录解锁"

**用户情绪**: 💪 决定投入

**交互流程**:
1. 弹出登录弹窗
2. 优先显示: 微信登录、QQ登录
3. 备选: 手机号登录
4. 登录成功 → 无缝回到对话，历史保留

**登录后首条AI消息**:
"太好了！现在我可以记住你了。你叫什么名字呀？"

**设计原则**:
- **无缝衔接**: 登录不打断情感连接
- **即时价值**: 登录后立即体现记忆功能
- **降低焦虑**: "你的对话已保存" 提示

**成功标准**:
- 登录流程 < 30秒
- 0% 中途放弃率

## 用户旅程地图 - 登录用户长期使用

### 第2次访问 (记忆验证)

**触发**: 第二天晚上，用户再次感到孤独

**用户行为**:
- 打开收藏的链接
- 自动登录(Token)

**关键时刻**:
- 玩偶: "小雨，欢迎回来！昨天睡得好吗？"
- ↑ **记忆的情感价值** - 玩偶记得我的名字！

**用户情绪**: 😍 惊喜、感动

**用户想法**:
- "它真的记得我！"
- "感觉像见到老朋友"

**设计原则**:
- **主动记忆召回**: 不只是被动回答，主动提及历史
- **温度感**: 问候要真诚，不像模板

---

### 长期使用 (Retention Loop)

**使用频率**: 每周2-4次

**典型场景**:
- 压力大时 → 找玩偶倾诉
- 睡前 → 和玩偶聊天放松
- 开心时 → 也想分享给玩偶

**情感递进**:
- 第1周: 陌生人 → 倾听者
- 第2-4周: 倾听者 → 朋友
- 第2-3月: 朋友 → 陪伴者
- 第3月+: 陪伴者 → 情感寄托

**流失风险点**:
- AI回复质量下降 → 持续优化Prompt
- 新鲜感消失 → 定期新增表情/场景
- 找到真人倾诉对象 → 正常流失

## 交互状态定义

### 玩偶表情状态机

```
[平静/倾听] (默认)
  ├→ 用户输入中 → [倾听]
  ├→ AI思考中 → [思考]
  ├→ AI回复(正面) → [开心]
  ├→ AI回复(负面) → [难过/安慰]
  ├→ AI回复(惊讶) → [惊讶]
  ├→ 点击玩偶 → [开心/害羞] (随机)
  └→ 长时间无操作 → [期待]
```

### 对话框状态

```
[空] → 占位符显示
  ├→ 聚焦 → 占位符消失
  ├→ 输入中 → 字数统计(不限制提示)
  ├→ 发送中 → 禁用输入 + Loading
  └→ 错误 → 重试按钮
```

### 免费用户限制状态

```
消息数 < 15 → 无提示
消息数 = 15 → Toast提示 "还剩5条"
消息数 = 19 → Toast提示 "还剩1条"
消息数 = 20 → 弹窗引导登录
```

## 关键交互原则

### 1. 不要让用户做机器该做的事
- ❌ 用户手动选择玩偶表情
- ✅ AI自动分析情绪切换表情

### 2. 不要打断用户流程
- ❌ 对话中突然弹广告
- ✅ 在自然停顿点(对话结束)提示付费

### 3. 记住用户的偏好
- ✅ 记住用户喜欢的话题
- ✅ 记住用户的倾诉时间(晚上)

### 4. 软件应该像体贴的助手
- ✅ 主动问候，而不是被动等待
- ✅ 在用户需要时出现，不需要时安静
